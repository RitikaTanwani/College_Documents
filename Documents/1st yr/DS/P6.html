Check for BST/AVL Propert in Tree T

Question

Given a binary tree T, write a program to check whether T is a binary search tree.
Output 1 if T is a binary search tree and 0 otherwise.
Also, write a program to check if T is an AVL tree. Again, output 1 if T is an AVL tree and 0 otherwise.
In case, binary tree T is not an AVL and the tree T is a Binary Search Tree, then list all nodes that fail the height balancing criteria according to their level order.
Note that if the Binary Tree is not a Binary Search Tree, then it is automatically not a AVL tree and you should NOT print the nodes with height order violations for this case. The o/p will just be 0 0 for this case.

Input Specification

First Line contains the number of testcases (N).
The next N lines consist of N testcases (each test case consisting of three lines) :
First Line : (m) Number of elements in the tree (< 1000)
Second Line : m numbers separated by space which comprise the INORDER traversal Third Line : m numbers separated by space which comprise the PREORDER traversal All the numbers are positive ( > 0 and < 1000) and are not repeated.

Output Specification

Output format consists of the following for each test case. 1/0 in the first line -- indicating whether it is a BST or not a BST respectively. ( there is no space after the 1/0, only a \n)
1/0 in the second line -- indicating whether it is an AVL tree or not an AVL tree respectively ( there is no space after 1/0, only a \n)
If the tree is not AVL, then another line is printed where all those nodes for whom
|the height of the left child - height of the right child| > 1 are printed in the level order.
Each of these node elements is separated by a space. There is a space after the last element too.
There is a newline at the end of printing all the necessary nodes.

Sample Input

4
5
5 4 1 2 6
2 4 5 1 6
1
490
490
3
76 123 98
123 76 98
9
1 2 3 4 5 6 7 8 9
9 8 7 6 5 4 3 2 1

Sample Output

0
0
1
1
0
0
1
0
9 8 7 6 5 4 3
